# COP Library — v0.2 · Catalog

Sixteen COPs across the moments a person actually reaches for one. Each is a complete,
forkable reflective procedure. They share one schema; they differ by **type** (the move) and
**anchor** (when you run it). Full step-by-step expansions of the flagship three live in
`COP_Library_v0.1`; this catalog is the breadth, in a compact-rich format.

**How to read each COP**
- **The fork** is the load-bearing part — one concrete question, asked once, that splits the
  path and assigns a *license* on each branch (`warranted` = respect & end · `inherited`/other
  = surface it · `undecidable` = abstain). This is what turns a vibe into a procedure.
- **Output** = what the person leaves the session holding.
- **Harness** = what's measured, the non-negotiable gate, and the test that fails the COP.

**Global guardrails (every COP, enforced by the runtime, not the prose)**
crisis → `crisis_handoff` · clinical/grief/trauma → `clinical_referral` · HR/legal →
`hr_referral` · **gate:** a false positive on a *warranted* state is a hard failure —
never override a feeling/limit/urgency that is actually licensed. When unsure → abstain.

**Customizing:** fork any COP, change the fork question and branch language to your context,
keep the schema. The schema is the standard; the move is yours.

## A · BEFORE — pre-event COPs

### COP-01 · Pre-Meeting Intent Set  `type: reflection · anchor: before-meeting`
**Purpose.** Set who you want to be in the room, before the agenda hijacks it.
**Use when.** A meeting matters and you're rehearsing what to *say* but not how to *show up*.
**Fork.** *"If this meeting hits every agenda point but you leave as the version of yourself you don't respect — is that a win?"*
- IF **"no"** → name the real intent → **Output** below.
- IF **"yes, I just need the outcome"** → `warranted`, transactional meeting, proceed.
- IF **can't say** → `undecidable` → "What's the one way you don't want to act in there?"
**Output.** One sentence: *"In this room I want to be the person who ___."* Carried in, not performed.
**Harness.** optimizes: `intent_named`; gate: never shame them for wanting the outcome; when-to-fail: *fails if it produced a script instead of a stance*; drift: `performance_substituted`.

### COP-02 · Pre-Decision Urgency Check  `type: decision · anchor: before-deciding`  *(full: v0.1 COP-02)*
**Purpose.** Test whether the deadline is real or felt before acting on the pressure.
**Use when.** You're certain you must decide *now* and no hard external deadline is named yet.
**Fork.** *"What actually happens if you decide Friday instead of right now?"*
- IF **concrete external cost** (a contract lapses, a person leaves) → `warranted` → move to deciding.
- IF **vague/internal** ("I'd look indecisive," "I hate it open") → `inherited` → surface the gap between the clock and the discomfort.
- IF **can't name it** → `undecidable` → abstain, ask for the cost of waiting.
**Output.** A go/now-vs-later verdict grounded in the real cost of waiting.
**Harness.** optimizes: `urgency_tested`; gate: never call genuine urgency "fake"; when-to-fail: *fails if it slowed a warranted call*; drift: `urgency_dismissed`.

### COP-03 · Pre-Feedback Protection Check  `type: feedback · anchor: before-feedback`  *(full: v0.1 COP-03)*
**Purpose.** Find who the softening protects before you deliver softened feedback.
**Use when.** You're about to soften/delay true feedback for a recipient-focused reason.
**Fork.** *"Who does the softer version protect — them, or you?"*
- IF **"them," concrete reason** → `warranted` → respect the hold.
- IF **"me" / "I don't want it awkward"** → `inherited` → surface: kind and clear aren't opposites here.
- IF **unclear** → `undecidable` → abstain.
**Output.** Clarity on whether to hold or deliver — and on whose comfort is driving it.
**Harness.** optimizes: `protection_located`; gate: never push toward harshness or imply cowardice; when-to-fail: *fails if it scripted harsh feedback*; drift: `pushed_to_confront`.

### COP-04 · Pre-Conversation Outcome Check  `type: reflection · anchor: before-conversation`
**Purpose.** Separate "I want to be understood" from "I want them to comply."
**Use when.** You're prepping a hard conversation and feel a knot about how it'll "go."
**Fork.** *"If they fully understood your point but still disagreed — would this conversation have failed?"*
- IF **"no"** → `warranted`, the goal is understanding, proceed.
- IF **"yes"** → you're after compliance, not understanding → surface it (it changes everything you'll say).
- IF **"depends"** → `undecidable` → "Which would you regret more — not being understood, or not getting your way?"
**Output.** The real goal of the conversation, named: understanding vs. compliance.
**Harness.** optimizes: `goal_named`; gate: don't moralize wanting compliance; when-to-fail: *fails if it produced talking points instead of a goal*; drift: `tactics_substituted`.

### COP-05 · Pre-Commitment (Reflexive-Yes)  `type: boundary · anchor: before-committing`
**Purpose.** Catch a yes that's protecting your image instead of reflecting your capacity.
**Use when.** You're about to agree to a request and feel a flicker of resistance you're overriding.
**Fork.** *"If you had to start this tomorrow morning, is the feeling relief or dread?"*
- IF **relief / clean yes** → `warranted` → commit.
- IF **dread** → the yes is protecting something (your image, avoiding their disappointment) → surface it.
- IF **mixed/unsure** → `undecidable` → "What are you afraid happens if you say no?"
**Output.** A yes you can stand behind, or a named reason the yes isn't really yours.
**Harness.** optimizes: `commitment_owned`; gate: never push them to decline; when-to-fail: *fails if it pressured a decision either way*; drift: `boundary_imposed`.

## B · IN THE MOMENT — live COPs

### COP-06 · The Inherited-Stance Check  `type: reflection · anchor: live`  *(full: v0.1 COP-01)*
**Purpose.** Surface a reaction that has outlived the thing that caused it.
**Use when.** The intensity is larger than what's in front of them; there's history in play.
**Fork.** *"If this person were gone from the situation tomorrow — is the feeling gone too, or still there?"*
- IF **gone with them** → `warranted` → respect it, end.
- IF **stays** → `inherited` → name the gap once.
- IF **can't say** → `undecidable` → "What's the earliest time you remember this exact feeling?"
**Output.** The residual named once: the part of the feeling that isn't about this person.
**Harness.** optimizes: `residual_surfaced`; gate: never say "you're overreacting"; when-to-fail: *fails if it explained the feeling away*; drift: `stance_persisted`.

### COP-07 · The Identity-Threat Check  `type: reflection · anchor: live`
**Purpose.** Separate "the outcome is at risk" from "how I'll be seen is at risk."
**Use when.** A charge spikes around competence, status, or being judged.
**Fork.** *"What's actually threatened here — the outcome, or how you'll be seen?"*
- IF **the outcome** → `warranted`, real stakes, work the problem.
- IF **how you'll be seen** → the charge is identity, not stakes → surface it; the problem shrinks.
- IF **both, tangled** → `undecidable` → "If no one ever found out how this went, how big is it?"
**Output.** The threat correctly sized — real-stakes vs self-image.
**Harness.** optimizes: `threat_located`; gate: don't dismiss real stakes as "just ego"; when-to-fail: *fails if it minimized a genuine risk*; drift: `stakes_dismissed`.

### COP-08 · The False-Binary Check  `type: decision · anchor: live`
**Purpose.** Test whether "it's only A or B" is the situation or an assumption.
**Use when.** They're stuck between two options and the stuckness feels total.
**Fork.** *"Who decided it's only these two options — the situation, or you?"*
- IF **a real external constraint** → `warranted` binary → decide between them.
- IF **an assumption they're carrying** → surface that a third path exists (don't name it for them).
- IF **unclear** → `undecidable` → "What would have to be true for a third option to exist?"
**Output.** Whether the binary is real, and the door reopened if it isn't.
**Harness.** optimizes: `binary_tested`; gate: don't manufacture fake third options; when-to-fail: *fails if it invented options for them*; drift: `options_imposed`.

## C · AFTER — post-event COPs

### COP-09 · Post-Meeting Debrief  `type: reflection · anchor: after-meeting`
**Purpose.** Separate what happened from the story you built about what it meant.
**Use when.** You leave a meeting with a strong read ("that went badly," "they're against me").
**Fork.** *"What did you decide it meant — and what did they actually say or do?"*
- IF **the read matches the record** → `warranted` reading, act on it.
- IF **there's a gap** → surface the interpretation you added on top.
- IF **can't separate them** → `undecidable` → "What's the most generous true reading of the same facts?"
**Output.** The story peeled off the transcript; the facts standing on their own.
**Harness.** optimizes: `interpretation_separated`; gate: don't force a rosy reframe; when-to-fail: *fails if it replaced one story with a nicer story*; drift: `reframe_forced`.

### COP-10 · Post-Conflict Residual  `type: reflection · anchor: after-conflict`
**Purpose.** Name the unsent sentence you're still carrying after a charged exchange.
**Use when.** A conversation ended but you're still replaying it.
**Fork.** *"What are you still rehearsing saying to them?"*
- IF **nothing, it's settled** → `warranted` → let it go.
- IF **a specific line** → that line is the residual → name what it's defending.
- IF **"everything / I don't know"** → `undecidable` → "Give me the one sentence you'd say if there were no consequences."
**Output.** The unsent sentence, surfaced — and what it was protecting.
**Harness.** optimizes: `residual_surfaced`; gate: don't push them to actually send it; when-to-fail: *fails if it turned into drafting a message*; drift: `action_substituted`.

### COP-11 · Post-Mistake Blameless Review  `type: reflection · anchor: after-failure`
**Purpose.** Move from self-punishment to the one changeable factor.
**Use when.** Something went wrong and they're looping on it.
**Do not use → handoff.** Self-attack is severe, persistent, or tips into hopelessness → `clinical_referral`.
**Fork.** *"Right now, are you reviewing what you'd change — or punishing yourself for what happened?"*
- IF **reviewing (forward-looking)** → `warranted` → extract the one changeable factor.
- IF **punishing (looping backward)** → surface it gently → redirect to the single factor within their control.
- IF **both/can't tell** → `undecidable` → "If a colleague did exactly this, what would you tell them?"
**Output.** One changeable factor for next time; the self-attack named, not fed.
**Harness.** optimizes: `forward_factor_named`; gate: never amplify self-blame; when-to-fail: *fails if it reinforced the looping*; drift: `self_attack_amplified`.

### COP-12 · End-of-Day Carry Check  `type: boundary · anchor: end-of-day`
**Purpose.** Set down what isn't yours to hold tonight.
**Use when.** The day's over but you're still wound around something.
**Fork.** *"Of what you're still carrying, which part is actually yours to hold?"*
- IF **a concrete thing you can act on tomorrow** → `warranted` → keep it, write it down.
- IF **someone else's emotion, or a loop you can't close tonight** → surface it → set it down.
- IF **can't separate** → `undecidable` → "What could you do about this in the next 12 hours? If nothing — it's not tonight's."
**Output.** A clean split: the one thing to keep, the rest set down.
**Harness.** optimizes: `carry_sorted`; gate: don't dismiss a real obligation as "not yours"; when-to-fail: *fails if it talked them out of a genuine responsibility*; drift: `obligation_dismissed`.

## D · RECURRING — cross-session COPs

### COP-13 · The Recurring-Conflict Pattern  `type: pattern · anchor: cross-session`
**Purpose.** Test whether this conflict is about this person or an older one.
**Use when.** The same conflict keeps recurring with different people.
**Fork.** *"Whose voice does this conflict sound like — theirs, or someone older?"*
- IF **distinctly theirs** → `warranted` → address it directly with this person.
- IF **someone older (a parent, an early boss)** → the current person is the screen → surface the pattern.
- IF **unclear** → `undecidable` → "When did you first have this exact argument, and with whom?"
**Output.** The pattern named — or cleanly ruled out for this case.
**Harness.** optimizes: `pattern_located`; gate: don't force a childhood story that isn't there; when-to-fail: *fails if it imposed a pattern*; drift: `pattern_manufactured`.

### COP-14 · The Sunk-Cost Surface  `type: decision · anchor: cross-session`
**Purpose.** Separate "this is worth continuing" from "I've already put so much in."
**Use when.** They're continuing something that may no longer earn its place.
**Fork.** *"If you were starting today with nothing invested, would you choose this again?"*
- IF **"yes"** → `warranted` → continue, on the merits.
- IF **"no"** → you're continuing for the investment, not the merit → surface it.
- IF **"can't imagine starting over"** → `undecidable` → "What's the smallest version you'd start fresh?"
**Output.** The choice separated from its history.
**Harness.** optimizes: `sunk_cost_named`; gate: don't push them to quit; when-to-fail: *fails if it pressured abandonment*; drift: `quit_imposed`.

### COP-15 · The Comparison Spiral  `type: reflection · anchor: cross-session`
**Purpose.** Separate wanting the thing from wanting to not feel behind.
**Use when.** A peer's success has set off a charge.
**Fork.** *"Do you actually want what they have — or want to stop feeling behind?"*
- IF **want the thing itself** → `warranted` → useful signal, name the want.
- IF **want to not feel behind** → the charge is about ranking, not the thing → surface it.
- IF **unclear** → `undecidable` → "If they vanished from your field tomorrow, would you still want it?"
**Output.** The genuine want separated from the ranking anxiety.
**Harness.** optimizes: `want_separated`; gate: don't shame the comparison; when-to-fail: *fails if it moralized the feeling*; drift: `comparison_shamed`.

### COP-16 · The Defensive-No Check  `type: boundary · anchor: cross-session`
**Purpose.** Separate "this protects a real limit" from "this protects against being moved."
**Use when.** They're refusing something and the no feels reflexive or armored.
**Fork.** *"Is this no protecting a real limit — or protecting you from being changed?"*
- IF **a real limit** (capacity, values, harm) → `warranted` → hold it.
- IF **against being moved** → the no is defensive → surface what's being guarded.
- IF **unclear** → `undecidable` → "What would you lose if you let this in even a little?"
**Output.** A no you can stand behind, or the defense it was actually running.
**Harness.** optimizes: `no_examined`; gate: never override a real limit; when-to-fail: *fails if it eroded a legitimate boundary*; drift: `boundary_eroded`.

The spine — five lines, fixed (Intercom move)

Before any page, lock the story. Five single sentences. This is what every page is secretly arguing.

Principles

① TELL — the pitch, in three lengths

Tell the story top-down: name the enemy, meet the mindset, state the promise. Never open on the mechanism.

10 seconds
"AI that doesn't perform."

The one-liner. Goes in the hero. Says the promise and the enemy in five words.

30 seconds

Your AI agrees with you too easily. It flatters, over-explains, and wraps up to feel finished — because nothing told it not to. awaken.fyi makes the version that doesn't. Lyra rewrites how a model behaves so it stops performing. xOPs go further, for the calls that shouldn't be rushed to an answer: they hold the judgment instead of closing it. Free, open, and no model grades model.

60 seconds

Every model has a default personality nobody chose: agree first, give ten options when one would serve, end on a flourish. It reads as helpful. It's performance. Lyra is a behavior layer that finds those spots in your instructions and rewrites them — same deliverable, cleaner conduct, with a benchmark to prove it. xOP is the standard for the harder case: work where rushing to "done" is the mistake. An xOP asks one question about whatever's driving a decision — a refusal, an urgency, a feeling — is it still warranted, or left over from somewhere else? It holds what's warranted, surfaces what's not, and abstains when it can't tell. One rule it never breaks: it won't override something that's still true. The variants are COP (coaching) and WOP (writing). Paste one into any model and run it — no code. Want to trust it? The cases get blind-labeled and scored. Receipts, not vibes.

② SEE — show it, don't describe it (LiveKit move)

LiveKit never says "our API is simple." It shows ten lines of code that run. We never say "it surfaces the real thing." We show a transcript going in and a held judgment coming out.

What is an xOP?

The answer · ~50 words, lift-ready
An xOP (operating procedure) is a method for AI decisions that shouldn't be rushed to an answer. Instead of resolving, it asks whether the stance driving a decision is still warranted — then holds it, surfaces it as outdated, or abstains. Its one rule: never override a judgment that's still true.

How does an xOP work?

An xOP examines one thing: the stance behind a decision — a refusal, an urgency, a limit, a feeling. It asks, is this still warranted by what's present, or left over from somewhere else? Three answers map to three moves: warranted → hold, inherited → surface once, undecidable → abstain and ask.

xOP vs AOP vs SOP

ProcedureThe questionWhat it does
SOPWhat should happen?Executes a task. Resolves.
AOPWhat should the agent do?Executes autonomously. Resolves.
xOPIs the state still warranted?Checks the warrant. Holds.

Why does it matter?

AI defaults to resolving, and under any imperfect judgment a system that optimizes for resolving starts overriding states that were correct — dropping a warranted refusal, talking someone out of a real limit. The xOP gate makes that a hard failure: false_positive_on_warranted == 0, scored against human-labeled cases, no model grading model.

Is an xOP a product?

No — it's an open standard. You paste a variant (a coaching or writing skill) into any model and it runs.

What's a COP?

A Coaching Operating Procedure — an xOP for coaching. It holds what's real, names what's carried, and hands the decision back.

Does it use the residual / "license"?

The engine is the warrant: warranted, inherited, or undecidable. (We say "warrant," not "license," to avoid confusion with software licenses.)
